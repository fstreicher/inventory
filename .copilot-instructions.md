# Copilot Instructions for Inventory Management App

## 🏗️ Project Overview
This is an Angular standalone inventory management application with Firebase/Firestore backend, featuring:
- **Google SSO Authentication** - Users can only sign in via Google
- **Client-side Encryption** - All sensitive data (box/item names and descriptions) is encrypted before storage
- **Offline-first Architecture** - App works offline and syncs when connection is restored
- **QR Code Integration** - Boxes can be scanned and accessed via QR codes
- **Reactive Architecture** - Heavy use of RxJS observables and Angular signals

## 📋 Code Style & Standards

### TypeScript/Angular Patterns
- **Always use private fields (`#`) instead of `private` keyword** for true privacy
- **Explicitly mark all public methods/properties with `public`** keyword (except for constructors)
- **Use Angular standalone components** (no NgModules)
- **Prefer dependency injection with `inject()` function** over constructor injection
- **Use TypeScript strict mode** - proper typing for all parameters and return values
- **Implement proper error handling** with typed error parameters (`error: any`)
- **Prefer `Array<T>` over `T[]` for array types** for consistency
- **Prefer self-closing tags** for components without children (e.g. `<my-component />`)

## 🔐 Security Requirements

### Authentication
- **Google SSO only** - no other authentication methods
- **Always verify user authentication** before any Firestore operations
- **Check user ownership** for all box/item operations
- **Clear encryption keys on logout** to prevent data leaks

### Data Encryption
- **All sensitive data MUST be encrypted** before storing in Firestore:
  - Box names and descriptions
  - Item names and descriptions
- **User IDs are NOT encrypted** (needed for queries)
- **Document IDs are NOT encrypted** (needed for references)
- **Use EncryptionService for all encryption/decryption operations**
- **Keys are derived from Google user UID + email** (consistent across sessions)

### Firestore Security
- **User-scoped data access** - users can only access their own boxes/items
- **Validate ownership** before any update/delete operations
- **Use proper Firestore security rules** (though not visible in client code)

## 🏛️ Architecture Patterns

### Services
- **FirestoreService**: Handles all CRUD operations with automatic encryption/decryption
- **EncryptionService**: Manages client-side encryption using Web Crypto API
- **AuthService**: Handles Google SSO authentication and logout
- **OfflineService**: Manages online/offline state detection
- **ThemeService**: Handles light/dark theme switching with signals

### Data Flow
1. **User authenticates** via Google SSO
2. **Encryption keys derived** from user credentials
3. **Data encrypted** on client before Firestore storage
4. **Data decrypted** on client after Firestore retrieval
5. **Automatic sync** when offline operations come back online

### Error Handling
- **Graceful offline handling** - queue operations and show appropriate messages
- **User-friendly error messages** for authentication and network issues
- **Console logging** for debugging but not sensitive data
- **Proper error typing** in RxJS operators

## 🔄 Common Patterns to Follow

### Service Method Pattern
```typescript
public getData(userId: string): Observable<Array<Data>> {
  const query = query(this.#collection, where('userId', '==', userId));
  return (collectionData(query, { idField: 'id' }) as Observable<Array<EncryptedData>>).pipe(
    switchMap(encryptedData => 
      from(Promise.all(encryptedData.map(item => this.#decrypt(item))))
    ),
    catchError(error => this.#handleError(error))
  );
}
```

### Component Observable Pattern
```typescript
public data$: Observable<Array<Data>> = this.#route.paramMap.pipe(
  switchMap(params => this.#service.getData(params.get('id')!)),
  startWith([]),
  catchError(error => {
    console.error('Error loading data:', error);
    return of([]);
  })
);
```

### Form Submission Pattern
```typescript
public onSubmit(): void {
  if (this.form.valid && !this.isSubmitting) {
    this.isSubmitting = true;
    const data = this.form.value;
    
    this.#service.addData(data).subscribe({
      next: () => {
        console.log('Success!');
        this.#router.navigate(['/success-route']);
      },
      error: (error: any) => {
        console.error('Error:', error);
        this.isSubmitting = false;
      }
    });
  }
}
```
